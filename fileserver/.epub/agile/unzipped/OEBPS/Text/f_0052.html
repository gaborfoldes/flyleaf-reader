8.2 Iteration C2: Adding a Page Layout

The pages in a typical website often share a similar layout—the designer will have created a standard template that is used when placing content. Our job is to modify this page to add decoration to each of the store pages.

So far, we’ve only made minimal changes to application.html.erb, namely to add a class attribute in ​​. As this file is the layout used for all views for all controllers that don’t otherwise provide a layout, we can change the look and feel of the entire site by editing just one file. This makes us feel better about putting a placeholder page layout in for now; we can update it when the designer eventually returns from the islands.

Let’s update this file to define a banner and a sidebar:

Apart from the usual HTML gubbins, this layout has three Rails-specific items. As mentioned previously, line 5 uses a Rails helper method to generate a <link> tag to our application’s stylesheet. Similarly, line 6 generates a <link> to our application’s scripts. Finally, line 7 sets up all the behind-the-scenes data needed to prevent cross-site request forgery attacks, which will be important once we add forms in Chapter 12, ​Task G: Check Out!​.

On line 12, we set the page heading to the value in the instance variable @page_title. The real magic, however, takes place on line 24. When we invoke yield, Rails automatically substitutes in the page-specific content—the stuff generated by the view invoked by this request. Here, this will be the catalog page generated by index.html.erb.

To make this all work, first rename the file application.css to application.css.scss, and then add the following:
